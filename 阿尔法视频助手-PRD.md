# 阿尔法视频助手 — 产品需求文档（PRD）

> 版本：v1.0 | 日期：2026-05-08 | 状态：已验收

---

## 一、文档概述

### 1.1 产品背景

短视频平台创作者在日常运营中面临大量重复性问题：审核申诉、播放量波动、收益结算、账号异常等。平台人工客服团队每天处理数千条咨询，其中 60% 以上为标准 FAQ 可覆盖的问题。现有客服体系存在以下痛点：

- **响应延迟**：高峰时段用户排队超过 30 分钟
- **回答不一致**：不同客服对同一问题的回复质量参差不齐
- **重复劳动**：客服人员大量时间消耗在"视频审核怎么申诉"等重复问题上
- **知识孤岛**：资深客服的经验无法沉淀为可复用的知识资产
- **创作者成长断层**：新手创作者缺乏系统性的视频制作指导和运营建议

### 1.2 产品愿景

打造一个**懂创作的 AI 客服助手**，不仅能 7×24 小时秒回常见问题，还能像创作伙伴一样帮助创作者提升内容质量、理解平台规则、规划变现路径。

### 1.3 目标用户

| 用户角色 | 核心需求 | 使用频率 |
|---------|---------|---------|
| 新手创作者 | 拍摄技巧、剪辑教程、平台规则入门 | 高（日活） |
| 成熟创作者 | 审核申诉、收益结算、流量分析 | 中（周活） |
| 平台客服人员 | 高效处理复杂工单、查看 AI 会话摘要 | 高（工作日） |
| 运营管理员 | 知识库维护、数据分析、质量监控 | 低（按需） |

## 二、产品目标

### 2.1 业务目标

| 指标 | 当前基线 | 目标值 | 达成时间 |
|------|---------|-------|---------|
| AI 自动解决率 | 0% | ≥ 70% | 上线后 3 个月 |
| 用户平均等待时长 | 15 分钟 | ≤ 30 秒 | 上线即达 |
| 客服人效提升 | 1x | 3x（处理复杂工单） | 上线后 1 个月 |
| 用户满意度评分 | 3.8/5 | ≥ 4.3/5 | 上线后 2 个月 |

### 2.2 产品目标

- **对话自然度**：用户难以区分 AI 和初级人工客服
- **知识覆盖率**：覆盖 80% 以上的常见创作者咨询场景
- **意图识别准确率**：≥ 90%（关键词兜底 + LLM 分类双保险）
- **系统可用性**：99.9% uptime，P99 延迟 < 2 秒

## 三、用户故事与功能需求

### 3.1 用户端（H5 / Web）

#### US-01：智能对话

> **作为** 创作者，**我想要** 通过自然对话描述我的问题，**以便** 快速获得准确的解答。

**验收标准：**
- 支持流式输出（SSE），逐字显示回复，消除等待焦虑
- 支持闲聊寒暄（你好、谢谢、再见），像朋友聊天一样自然
- 检测用户情绪（焦急、愤怒），先共情再给方案
- 同一会话内保持上下文记忆，支持追问和补充描述

#### US-02：意图识别与知识库检索

> **作为** 创作者，**我想要** 系统能自动理解我的问题类型，**以便** 不用手动选择分类就能得到精准回答。

**验收标准：**
- 支持 8 大类意图分类：闲聊、视频创作、账号问题、内容运营、变现相关、互动社交、平台功能、其他
- 知识库命中时，以自然语言包装 KB 内容返回，而非机械复制粘贴
- 知识库未命中时，友好引导用户换一种描述方式或转人工
- 回复末尾给出明确的下一步引导

#### US-03：转人工

> **作为** 创作者，**我想要** 在 AI 无法解决时无缝转接人工客服，**以便** 复杂问题不被搁置。

**验收标准：**
- 支持用户主动输入"转人工"触发
- AI 自动生成会话摘要，让人工客服无需翻聊天记录
- 转接过程显示排队位置和预计等待时间
- 人工接入后显示客服身份标识（姓名/工号）
- 支持结束人工对话返回 AI 模式

#### US-04：推荐问题

> **作为** 新手创作者，**我想要** 看到推荐的热门问题，**以便** 快速了解平台功能和创作技巧。

**验收标准：**
- 提供 6 个"猜你想问"按钮，覆盖视频创作高频问题
- 提供快捷分类入口（拍摄技巧、涨粉运营、收益提现等）
- 推荐问题内容随版本迭代持续优化

### 3.2 客服工作台（Web 管理端）

#### US-05：会话管理

> **作为** 客服人员，**我想要** 查看待处理会话列表和 AI 生成的摘要，**以便** 快速接手复杂工单。

**验收标准：**
- 展示待处理/进行中会话列表，按优先级排序
- 每条会话显示 AI 意图分类、情绪标签、对话摘要
- 支持认领会话、查看完整对话历史
- 支持快捷回复模板

#### US-06：工单系统

> **作为** 客服人员，**我想要** 将需要跨部门处理的问题创建为工单，**以便** 跟踪处理进度。

**验收标准：**
- 支持一键从会话创建工单（自动填充摘要）
- 工单支持设置优先级（urgent/high/medium/low）
- 支持分类归档（账号/内容/变现/社交/平台/其他）
- 工单状态流转：待处理 → 处理中 → 已解决 → 已关闭

### 3.3 管理后台

#### US-07：知识库管理

> **作为** 运营管理员，**我想要** 维护和更新知识库内容，**以便** AI 能回答最新的平台规则。

**验收标准：**
- 支持批量导入 FAQ（JSON/Excel）
- 支持在线编辑知识条目（标题、正文、分类、相似问题）
- 支持知识库检索质量测试（输入问题 → 查看命中结果和相似度）
- 向量索引自动重新构建

#### US-08：数据看板

> **作为** 运营管理员，**我想要** 查看客服系统的核心数据指标，**以便** 持续优化服务质量。

**验收标准：**
- 核心指标卡片：今日会话数、AI 解决率、转人工率、平均满意度
- 意图分布饼图
- 满意度趋势折线图（按天/周）
- 知识库命中率与未匹配问题 TOP 10
- 支持导出报表

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 首字响应时间 | P50 < 500ms，P99 < 1500ms |
| 知识库检索延迟 | P99 < 200ms |
| 并发会话数 | 支持 1000+ 并发 |
| SSE 流式吞吐 | 单连接稳定传输 > 10 分钟 |

### 4.2 安全要求

- API Key / 数据库密码等敏感信息必须从环境变量读取，禁止硬编码
- 用户对话内容存储加密（AES-256）
- 敏感信息自动脱敏（手机号、身份证号、银行卡号）
- 请求限流：单用户 30 次/分钟
- CORS 白名单控制（生产环境）

### 4.3 可维护性

- 核心模块（Agent 循环、意图分类、工具调用）可独立测试
- 知识库支持热更新，无需重启服务
- 结构化日志输出（JSON 格式），接入 ELK
- 健康检查端点：`/health`

### 4.4 兼容性

- 服务端：Python 3.11+
- 向量模型：BAAI/bge-small-zh-v1.5（512 维）
- LLM：Anthropic Claude Sonnet 4.6 / DeepSeek V4
- 数据库：SQLite（开发）/ PostgreSQL 15+（生产）
- 向量库：Chroma（开发）/ Milvus（生产）
- 前端：Chrome 90+, Safari 15+, 移动端 WebView

### 核心技术选型理由

| 技术 | 选择理由 |
|------|---------|
| FastAPI | 原生异步支持 + SSE Streaming + 自动 OpenAPI 文档 |
| Anthropic Claude | 中文理解能力强，工具调用（Tool Use）原生支持 |
| Chroma | 轻量级向量库，开发环境零配置，支持嵌入式部署 |
| SentenceTransformers | BGE 中文模型在短文本相似度上表现优异 |
| SQLAlchemy 2.0 Async | 异步 ORM，开发期用 SQLite 免配置，生产切 PostgreSQL |

### Agent 核心循环

```
用户输入 → 意图分类 → 知识库检索 → 工具调用(可选) → 生成回复 → 流式输出
                ↑                                          │
                └────────── 记忆更新 + 上下文维护 ←────────┘
```

## 六、数据模型

```
Conversation（会话）
├── id: UUID
├── user_id: str
├── user_type: enum(creator, normal, agent)
├── user_name: str
├── status: enum(ai_processing, human_processing, resolved, closed)
├── satisfaction_score: int (1-5)
├── summary: str
└── created_at / updated_at

Message（消息）
├── id: UUID
├── conversation_id: FK → Conversation
├── sender_type: enum(user, ai, human_agent, system)
├── content: str
├── intent: str
└── created_at

Ticket（工单）
├── id: UUID
├── conversation_id: FK → Conversation
├── title: str
├── category: enum(account, content, monetize, social, platform, other)
├── priority: enum(urgent, high, medium, low)
├── status: enum(pending, processing, resolved, closed)
├── summary: str
└── created_at / updated_at

KnowledgeEntry（知识条目）
├── id: str
├── question: str
├── answer: str
├── category: enum(account, content, monetize, social, platform, video_create)
├── similar_questions: list[str]
└── embedding_vector: float[512]
```

## 八、风险与应对

| 风险 | 等级 | 应对措施 |
|------|------|---------|
| LLM 生成内容不可控（幻觉） | 高 | 知识库检索优先于 LLM 自由生成；敏感话题硬约束 |
| 知识库内容过时 | 中 | 运营定期更新机制；知识条目增加有效期字段 |
| 高并发下 LLM API 限流 | 中 | Redis 缓存高频问题回复；Demo 模式兜底 |
| BGE 模型对长文本效果下降 | 低 | 知识条目控制在 300 字以内；分段索引策略 |

## 九、附录

### A. 意图分类体系

```
chitchat    → 问候 / 感谢 / 告别 / 夸奖 / 吐槽
video_create → 拍摄技巧 / 剪辑教程 / 音乐配乐 / 文案写作 / 封面设计
account     → 登录失败 / 账号封禁 / 账号找回 / 实名认证
content     → 发布失败 / 审核申诉 / 限流问题 / 上热门 / 版权咨询
monetize    → 收益提现 / 创作者计划 / 直播打赏 / 电商带货
social      → 评论管理 / 举报用户 / 粉丝增长 / 私信问题
platform    → 新功能咨询 / 设置隐私 / 青少年模式
other       → 吐槽建议 / 无效输入
```

### B. 转人工触发条件

1. 用户明确输入"转人工"关键词
2. AI 连续 2 轮无法给出满意回答（用户反馈不满意）
3. 涉及资金纠纷（收益未到账超 3 个工作日）
4. 账号永久封禁申诉
5. 用户情绪值触发阈值（连续负面表达）
